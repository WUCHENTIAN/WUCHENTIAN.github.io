1. Binary Search Tree

二叉搜索树具有很良好的 插入/删除/搜索 性质

这是传统的线性表做不到的

1.1. 递归式定义

​ A binary search tree is a binary tree that may be empty. A nonempty binary search tree satisfies the following properties:

  • Every element has a key (是一个包含序关系的值，用于比较) and no two elements have the same key (因此所有的 key 值都要是不重复的)

  • The keys(if any) in the left subtree of the root are smaller than the key in the root.

  • The keys(if any) in the right subtree of the root are larger than the key in the root.

  • The left and right subtrees of the root are also binary search trees.

1.2. 带索引的二叉搜索树

  • An indexed binary search tree is derived from an ordinary binary search tree by adding the field leftSize to each tree node. （添加一个左子树节点数的缓存）

  • Value in Leftsize field = number of the elements in the node’s left subtree +1

1.3.5. 删除

It is necessary to adjust the binary search tree after deleting an element, so that the tree remained is still a binary search tree.There is three cases for deleting node p :

  • P is a leaf

  • P has exactly one nonempty subtree

  • P has exactly two nonempty subtrees

1.4. Binary Search Tree 的复杂度

  1. 二叉搜索树以上的所有操作都和二叉搜索树的深度有关，所以在生成二叉树的时候我们需要保证二叉搜索树的平衡性，(如果一开始输入最小的，树严重失衡，如果一开始输入中等，树基本平衡)
  2. Best Case: \(O(\log_2n)\)
  3. Worst Case: \(O(n)\)

2. AVL Tree

2.1. 定义

  • An AVL Tree is a Binary Search tree
  • Every node satisfies \(|h_L -h_R |\leq 1\) where \(h_L\) and \(h_R\) are the heights of \(T_L\) (left subtree) and \(T_R\) (right subtree), respectively.

2.2.

The height of an AVL tree with n elements is O(log2 n), so an n-element AVL search tree can be searched in O(log2 n) time.
